Create a New Web Site (Choose Virtual Directories)

You have chosen to use $$WebsiteTemplate$$ as a template from which to create $$WebsiteName$$. On this page, you will chose which virtual directories from the existing Web site you want to be available on your new site.

Certain virtual directories maybe marked as read-only. (Why?) This means that if you select the virtual directory for this new Web site, you will never be able to delete it. If you do not copy a read-only virtual directory now, you will not be able to add one later, either. (You can add any other virtual directory at a later stage, however).

Read-only virtual directories are typically server-side scripts that provide configuration options for your device. The WebAdmin tool you are using is one such example. Bringing this read-only virtual directories into your new Web site will allow anyone on your device to potentially access them (though they can also be protected with other authentication mechanisms). Not selecting them may impair the user experience on the Web site. If you are not sure if you need this or any other virtual directory on the website you are creating, consult the documentation. If you are still unsure about whether to copy a virtual directory or not, for highest security Microsoft recommends not copying it.
